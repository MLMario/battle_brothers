**1. `applyFilter()` function (D-05, D-06, D-09, D-10):**

```javascript
  // ── Phase 5 D-05/D-06/D-09/D-10: filter pipeline (drives render + count + empty) ──
  function applyFilter() {
    // D-10: clear stale openId before re-render destroys/recreates DOM nodes
    openId = null;

    // D-06: case-insensitive substring match on name; empty query → cloned full list
    const q = query.toLowerCase().trim();
    filtered = q
      ? allBgs.filter(function (b) { return b.name.toLowerCase().includes(q); })
      : allBgs.slice();   // CLONE — do NOT share reference with allBgs

    // D-05: direct pipeline (Phase 6 will insert applySort() between filter and render)
    renderList(filtered);
    updateCount(filtered.length, allBgs.length);

    // D-09: empty-state rule — zero matches AND data is loaded
    setEmpty(filtered.length === 0 && allBgs.length > 0);
  }
```

Style notes:
- Use `function () { … }` arrow-free callbacks to match the existing code style in `app.js` (e.g. lines 71, 120, 224, 410). The codebase uses arrow functions sparingly; named-function callbacks are the dominant pattern.
- `allBgs.slice()` and `[...allBgs]` are equivalent for cloning — pick whichever matches the surrounding code's idiom (the codebase uses neither today; either is acceptable).

**2. `wireControls()` function (D-07, D-13, D-14, D-20):**

```javascript
  // ── Phase 5 D-07/D-13/D-14: wire #search input → debounced applyFilter ──
  function wireControls() {
    const searchEl = document.getElementById('search');
    if (!searchEl) return;   // defensive guard — matches app.js convention (CODEBASE.md §2)

    let searchTimer = null;
    searchEl.addEventListener('input', function onSearchInput() {
      clearTimeout(searchTimer);
      searchTimer = setTimeout(function onSearchDebounce() {
        query = searchEl.value;
        applyFilter();   // applyFilter handles openId = null itself (D-10)
      }, 180);
    });
    // D-13: no keyboard handlers, no change/keyup wiring — type="search" native input event covers all clear paths (CODEBASE.md §4.6).
    // D-14: never set searchEl.disabled — input is always typable; applyFilter no-ops against empty allBgs.
  }
```

Critical:
- Debounce window is exactly **180ms** (D-07, mockup-locked, do not change).
- ONLY listen to `'input'` — do NOT add `keyup`, `change`, `keydown`, or `keypress` handlers. The native clear "×" of `type="search"` fires `input` per spec (CODEBASE.md §4.6).
- `searchTimer` is a closure-scoped `let` inside `wireControls()` — do NOT promote to module scope.
- Do NOT touch `#list.scrollTop` (D-08).
